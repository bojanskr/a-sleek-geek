gRPC is a modern open source high performance Remote Procedure Call (RPC) framework that can run in any environment. It can efficiently connect services in and across data centers with pluggable support for load balancing, tracing, health checking and authentication. It is also applicable in last mile of distributed computing to connect devices, mobile applications and browsers to backend services.

The main usage scenarios

  • Efficiently connecting polyglot services in microservices style architecture
  • Connecting mobile devices, browser clients to backend services
  • Generating efficient client libraries

The story behind gRPC

gRPC was initially created by Google, which has used a single general-purpose RPC infrastructure called Stubby to connect the large number of microservices running within and across its data centers for over a decade. In March 2015, Google decided to build the next version of Stubby and make it open source. The result was gRPC, which is now used in many organizations outside of Google to power use cases from microservices to the “last mile” of computing (mobile, web, and Internet of Things).

Short Overview

In gRPC, a client application can directly call a method on a server application on a different machine as if it were a local object, making it easier for you to create distributed applications and services. As in many RPC systems, gRPC is based around the idea of defining a service, specifying the methods that can be called remotely with their parameters and return types. On the server side, the server implements this interface and runs a gRPC server to handle client calls. On the client side, the client has a stub (referred to as just a client in some languages) that provides the same methods as the server.

An interesting part:

Official gRPC website overview

gRPC clients and servers can run and talk to each other in a variety of environments – from servers inside Google to your own desktop – and can be written in any of gRPC’s supported languages. So, for example, you can easily create a gRPC server in Java with clients in Go, Python, or Ruby. In addition, the latest Google APIs will have gRPC versions of their interfaces, letting you easily build Google functionality into your applications.

Protocol Buffers

gRPC utilizes PROTOCOL BUFFERS – Google’s language-neutral, platform-neutral, extensible mechanism for serializing structured data – think XML, but smaller, faster, and simpler.

A good place to start is Google’s official documentation at:

https://developers.google.com/protocol-buffers/

if you want to find out more.

GitHub Repositories

So much for the introduction. Now let’s go to GitHub and see something more practical.

First let’s list the repos that are important for us here:

  • https://github.com/grpc/grpc
    • this is the main repo from where you can start, it contains tons of useful info like links to docs, chats, information on how to start using gRPC etc
    • this repository contains source code for gRPC libraries implemented in multiple languages written on top of a shared C core library src/core.
  • https://github.com/grpc/grpc.io
    • the frpc.io website repo, there’s also instructions on how to get the docs for the submodules here as well

Now the repos for the different implementations / different technologies:

  • https://github.com/grpc/grpc-node
    • for node: npm install grpc
  • https://github.com/grpc/grpc/tree/master/src/csharp
    • gRPC C#
    • there is also a new implementation for .net core 3.0 and above and that repo is at: https://github.com/grpc/grpc-dotnet
    • You can read more on Introduction to gRPC on .NET
  • https://github.com/grpc/grpc/tree/master/src/python/grpcio
    • for Python installation: pip install grpcio
    • note: Python version 3.5 and greater are supported
